K-Scaffold: JS Library

These are the javascript functions and members included in the K-Scaffold. They can be all called as properties of k, such as k.getAttrs.

Members

(constant) attributesProxy :object

The attributes object that is generated via createAttrProxy(), and used in k.getAttrs() & k.getAllAttrs(). This is a proxy for the basic attributes object passed to the callback in the sheetworker [getAttrs()](https://wiki.roll20.net/Sheet_Worker_Scripts#getAttrs.28attributeNameArray.2C_callback.29).
Type:
  • object
Properties:
Name Type Attributes Description
name_of_attribute string | number <optional>
 Your attribute values are stored in the attribute proxy under their attribute names. You can reference the current value of the attribute by calling it directly from the proxy (e.g. `attributes.my_attribute`). This will return either the original value if no changes have been made, or the updated value if changes have previously been applied. Storing an updated value is done by simple assignment (e.g. `attributes.my_attributes = 5`).
set function Applies any updates to the sheet. Also applies any repeating section reorders. This uses ES6 destructuring function argument assignment so all arguments should be passed in as properties of a single object with the indicated keys (e.g. `attributes.set({vocal:true,callback:() => log('attributes were set')})`)
Properties
Name Type Attributes Description
vocal boolean <optional>
 set will not be silent. Inverts the standard behavior of setAttrs options object
callback function <optional>
 A callback to be invoked once the setAttrs is completed
attributes attributesProxy <optional>
 The instance of the attributeProxy to use for further set operations
sections object An object containing the idArrays for each repeating section
casc object as the casc property
attributes object The original attribute values from the getAttrs call
updates object Any changes that are made by the script are stored here
repOrders object any changes to repeating section orders stored here
queue Array.<string> queue of attributes affected to iterate through. Only used if the whole proxy is passed to attributes.set()
casc object The details of how to handle all attributes
processChange function function to iterate through attribute changes for default handling

initialFunction function Calls functions that are only triggered when an attribute is the triggering event
getCascObj function Gets the appropriate cascade object for a given attribute or action button

expandedCascade

Type:
  • object
Properties:
Name Type Attributes Description
itemName object Object with details of how to handle the attribute or button. Attribute names are used as the key with a prefix of `attr_` and button names are used as the key with a prefix of `act_`. Repeating attributes/buttons should use $x or $X in place of an explicit ID. This will be expanded by the scaffold in response to an event.
Properties
Name Type Attributes Description
name string The name of the attribute without the `act_` or `attr_` prefix.
type string The type of the attribute. Autopopulated from the `type` property from the attribute definition in the PUG.
listener string <optional>
 the listener text to be used in a [roll20 event listener](https://wiki.roll20.net/Sheet_Worker_Scripts#Event_listener).
defaultValue string | number <optional>
 The default value of the attribute. Not used for buttons. Autopopulated from the `value` property of the PUG definition of the attribute.
itemName.listenerFunc string <optional>
 Name of function to call from the [roll20 event listener](https://wiki.roll20.net/Sheet_Worker_Scripts#Event_listener). Is used as a string to call the function from the funcs object.
itemName.initialFunc string <optional>
 Name of function to call only when the attribute is the change event that triggered the listener.
itemName.calculation string <optional>
 Name of function to call to calculate the value of this attribute.
itemName.triggeredFuncs Array.<string> <optional>
 Array of function names that should be called any time this attribute is changed or affected.
itemName.yourPropertyName * <optional>
 Additional properties can be added to this setup via the K-Scaffold's PUG interface.
